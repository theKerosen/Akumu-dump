This repository serves as a technical case study in retrieving source code from compiled executables. It contains the raw Game Maker Language (GML) scripts, object definitions, and asset metadata extracted from the game "**Akumu**".

> Static analysis of `oCredits` revealed persistent string literals referencing an entity known as **"Asset Bakery."**
> Despite these internal references, this entity **does not appear** in the game's public-facing credits sequence.
> *   Did they do the work?
> *   Were they erased from history?
> The bytecode knows the truth, even if the developer refuses to admit it.

## Technical Overview

The primary objective was to perform static analysis on a compiled PE32 Waindows executable within a strict **Arch Linux** environment.

The decompilation pipeline involved:

1.  **Cross-Platform Execution:** Leveraging **Wine and Proton** to bridge the compatibility layer for Windows-native analysis tools.
2.  **Archive Unpacking:** Utilizing **QuickBMS** with custom script definitions to parse the proprietary IFF-chunk structure of the `data.win` container.
3.  **Bytecode Translation:** Deserializing the Game Maker Virtual Machine (GMVM) instruction sets into human-readable GML.

## Code "Smells"

Reverse engineering reveals not just the *how*, but the *why*â€”and sometimes, the "why" is terrifying. Below are specific examples of technical debt found within the binary.

### 1. The "Wall of Ifs"
In `oBasementFloor2`, room management is handled via a hardcoded O(n) check that lists every single room manually. If the developer adds one room and forgets to update this list, the object self-destructs.

```gml
// Found in: src/objects/oBasementFloor2_Step_0.gml
if (room == rBasement9 || room == rBasement10 || room == rBasement11 ... ) {
    // Logic
} else {
    instance_destroy(); // The nuclear option
    show_debug_message("destroyed");
}
```

### 2. The "Creepy Key" Serialization
The save system (`scrSaveGame`) avoids standard structs or arrays in favor of manually mapping variables to INI keys that form... sentences?

```gml
// Found in: src/scripts/scrSaveGame.gml
ini_write_real("stage1_1", "you", sink);
ini_write_real("stage1_1", "shouldn't", toilet);
ini_write_real("stage1_1", "be", paper);
ini_write_real("stage1_1", "here", all_);
```
*Verdict: Efficient? No. Unsettling? Yes.*

### 3. Debug Artifacts
The developer left a functional debug backdoor in the production build. *wowie.*
```gml
// Found in: src/objects/oGlobal_Step_0.gml
if (keyboard_check(vk_subtract)) {
    room_goto(rDebug); // Instantly teleports the player
}
```

### 4. The "Nervous Furniture"
Static objects (Tables, Bookshelves, Beds) contain code in their `Step` eventâ€”which runs 60 times per secondâ€”solely to update their depth sorting.

```gml
// Found in: src/objects/oTable2_Step_0.gml
// This table is stationary. It will never move. 
// Yet, it recalculates its Z-ordering every single frame.
if (normalDepth)
    depth = -bbox_bottom;
```

### 5. Callback Purgatory
Instead of using a Timeline or a proper State Machine for cutscenes, the code relies on a fragile nesting of anonymous functions inside custom `Delay()` scripts.

```gml
// Found in: src/objects/oFinalDemoMonologue_Step_0.gml
Delay(200, function() {
    havingDialogue++;
    OpenTextBox(...);
    Delay(2000, function() {
        oTextBox.pages = -4;
        OpenTextBox(...);
        Delay(2000, function() { // We have to go deeper
            // Hope the frame rate didn't dip, 
            // otherwise this sync is ruined forever.
        });
    });
});
```

### 6. Variable Laundering
In `scrSaveGame`, global variables are not saved directly. Instead, they are assigned to local variables named... numbers?

```gml
// Found in: src/scripts/scrSaveGame.gml
var zero = global.dialogue;
var one = global.basementpuzzlecomplete;
var two = global.dialogue2;

// Why save "global.dialogue"? 
// Let's save "zero" into key "0" instead.
ini_write_real("stage1", "0", zero);
```

## Project Structure

*   `src/objects/`: Core entity logic and state machines.
*   `src/scripts/`: Global helper functions.
*   `src/rooms/`: Level layout definitions.
*   `src/assembly/`: Assembly code.
*   `graphics/`: Graphics assets.
*   `sounds/`: Audio assets.
*   `strings/`: Binary strings.
